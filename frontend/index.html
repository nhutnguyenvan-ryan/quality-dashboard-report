1.2 Data Pipeline — Luồng Fetch Bảo mật
1

Xác thực Service Account (Server-side)

Private Key của Service Account được lưu trong biến môi trường (GSHEETS_PRIVATE_KEY), không bao giờ expose ra client. Backend ký JWT và đổi lấy Access Token qua oauth2.googleapis.com/token.

2

Fetch dữ liệu theo Named Ranges

Mỗi sheet có named ranges riêng (QA_WEEKLY, APPEAL_DATA, LEAKAGE_DATA…). Backend gọi batchGet để lấy nhiều range cùng lúc, giảm số API call.

3

Transform & Normalize

Raw values từ sheet (mảng 2D) được parse thành JSON chuẩn. Áp dụng: type coercion, null handling, week/month key normalization (W18 → 2025-W18).

4

Cache & Serve

Kết quả được cache với TTL 15 phút. Manual refresh (nút "Refresh Data") buộc invalidate cache. Frontend lấy data qua internal API endpoint (/api/qa-data).

5

CSV Upload Path (Parallel)

File CSV từ user được validate schema, parse và merge vào data store theo cùng chuẩn normalize. Không ghi ngược lại Google Sheets — chỉ lưu session-local.

Bảo mật: Spreadsheet ID và Service Account credentials không bao giờ được nhúng vào frontend bundle. Tất cả API calls tới Google đều đi qua backend proxy để tránh lộ credential.

2.3 Period Range Normalization
Weekly Mode
Format key: W{number}
Range picker: dropdown từ W1 → W52
Default hiển thị: 5 tuần gần nhất
Sort: ascending theo week number
Monthly Mode
Format key: YYYY-MM
Range picker: multi-select tháng (Jan → Dec)
Default hiển thị: 3 tháng gần nhất
Probation tab: ẩn hoàn toàn khi Monthly
